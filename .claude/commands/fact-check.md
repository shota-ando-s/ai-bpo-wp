# /fact-check — 記事ファクトチェック

対象ファイル: $ARGUMENTS

---

記事内の事実的主張を抽出・検証し、誤情報・要確認箇所をレポートする。

---

## ステップ1: ファイル読み込み

`$ARGUMENTS` が空の場合、対象ファイルのパスをユーザーに確認する。

Read ツールでファイルを読み込み、以下を把握する:
- 記事のテーマ・対象読者
- フロントマター（title / categories / tags）
- 本文全体

---

## ステップ2: 事実的主張の抽出

本文から以下のカテゴリに該当する主張をすべて抽出してリスト化する。

### 抽出対象

| カテゴリ | 例 |
|----------|-----|
| 数字・統計 | 「市場規模は〇兆円」「XX%が〜」「〇〇社中△△社が」 |
| 日時・年号 | 「2023年に〜が始まった」「〇月〇日に発表」 |
| 固有名詞 | 企業名・製品名・人名・法律名・機関名 |
| 因果関係 | 「〇〇のため〜が起きた」「〇〇が原因で〜」 |
| 定義・説明 | 「〇〇とは〜のことです」「〇〇は〜の仕組みで動く」 |
| 引用・発言 | 「〇〇氏は〜と述べた」「〇〇社の発表によると」 |
| 比較・ランキング | 「日本は〇位」「世界最大の〜」「〇〇より△△が多い」 |
| 法律・制度 | 「〇〇法では〜が義務付けられている」 |

---

## ステップ3: 並行検証（Agentツールで2エージェントを同時起動）

抽出した主張を前半・後半に分割し、2エージェントを並行起動して検証する。

### 検証エージェントへの共通指示

以下の手順で各主張を検証する:

1. WebSearch で主張の正確性を確認する
2. 一次情報源（公的機関・公式発表・査読論文・大手メディア）を優先する
3. 各主張に対して以下の判定を下す:

**判定基準**

- ✅ **確認済み**: 信頼できる情報源で裏付けが取れた
- ⚠️ **要注意**: 古い情報・情報源が弱い・細部が不正確・解釈に幅がある
- ❌ **誤り**: 明確に事実と異なる
- ❓ **確認不能**: 検索しても裏付けが取れない（存在しない可能性）

各主張について出力:
```
主張: [記事内の原文]
判定: ✅/⚠️/❌/❓
根拠: [確認した情報源・URL]
補足: [正確な数値・最新情報・注意点など]
```

---

## ステップ4: ファクトチェックレポート提示

2エージェントの結果を統合し、以下のフォーマットで提示する。

```
## ファクトチェックレポート: [記事タイトル]

### サマリー
✅ 確認済み: X件
⚠️ 要注意: X件
❌ 誤り: X件
❓ 確認不能: X件

---

### ❌ 誤り — 要修正

**[主張の概要]**
- 記事の記述: 「[原文]」
- 正確な情報: [正しい内容]
- 出典: [URL]

---

### ⚠️ 要注意 — 修正または注記推奨

**[主張の概要]**
- 記事の記述: 「[原文]」
- 問題点: [古い情報 / 出典なし / 解釈に幅がある など]
- 推奨対応: [更新すべき数値 / 注記を加える / 表現を和らげる など]
- 参考: [URL]

---

### ❓ 確認不能 — 出典明記または削除推奨

**[主張の概要]**
- 記事の記述: 「[原文]」
- 状況: [裏付けが見つからない理由]
- 推奨対応: [出典を明記する / 「〜とも言われています」と断定を避ける / 削除]

---

### ✅ 確認済み

[主張のリスト（簡略表示）]
```

---

## ステップ5: 修正対応確認

レポート提示後、ユーザーに確認する:

「修正しますか？」
- **自動修正**: ❌誤りと⚠️要注意の箇所を一括修正する
- **個別修正**: 指定した箇所だけ修正する
- **修正案提示のみ**: ファイルは変更せず、修正テキストだけ出力する
- **このままでOK**: 修正しない

---

## ステップ6: 修正実行（選択時のみ）

ユーザーの選択に応じて修正を行う。

修正前にバックアップを作成する:
```bash
cp articles/元ファイル名.md articles/元ファイル名.md.bak
```

修正方針:
- ❌ 誤り: 正確な情報に書き換える。数値は出典URLをリンクで併記する
- ⚠️ 要注意（古い情報）: 最新データに更新し「（〇〇年時点）」と明記する
- ⚠️ 要注意（断定が強い）: 「〜とも言われています」「〜の傾向があります」など表現を和らげる
- ❓ 確認不能: 「（要出典）」と注記するか、該当箇所を削除する

修正完了後:
「修正が完了しました。投稿しますか？」
投稿する場合:
```bash
python3 post.py articles/ファイル名.md
```
