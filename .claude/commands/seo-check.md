# /seo-check — 既存記事のSEOチェック＆リライト

対象ファイル: $ARGUMENTS

---

既存記事のSEOを診断し、問題点を報告する。必要に応じてリライトも行う。

---

## ステップ1: ファイル読み込み

`$ARGUMENTS` が空の場合、対象ファイルのパスをユーザーに確認する。

ファイルパスが指定されたら Read ツールで記事を読み込む。
フロントマター（title / description / slug / categories / tags）と本文を把握する。

---

## ステップ2: SERP・競合調査（並行）

記事のメインキーワードを推定し、以下の2エージェントを並行起動する。

### SERPチェックエージェントへの指示

記事タイトルとメインキーワードを元に WebSearch で以下を確認する:

1. 現在の検索意図（情報収集型/比較型/購買型）
2. SERP上位のコンテンツ形式（リスト/Q&A/ハウツー/定義型）
3. 強調スニペットの有無と形式
4. People Also Ask（PAA）に出ている質問を10個以上抽出
5. 上位記事の推定文字数レンジ

出力: 検索意図・推奨フォーマット・PAA質問リスト・推定文字数レンジ

### 競合ギャップエージェントへの指示

WebSearch で上位3記事の構成を確認し、対象記事と比較する:

1. 競合がカバーしているが対象記事が触れていないトピック
2. 競合が引用しているデータ・統計（対象記事との差）
3. FAQセクションの有無・質問数
4. 内部リンク・外部リンクの充実度

出力: 競合との差分リスト（追加すべきトピック・データ・FAQ）

## ステップ4: 診断レポート提示

以下の形式でまとめて提示する:

```
## SEO診断レポート: [記事タイトル]

### スコア
✅ 合格: X項目
⚠️ 改善推奨: X項目
❌ 要修正: X項目

### 優先度HIGH — 要修正
1. [問題点]: [具体的な修正案]
2. [問題点]: [具体的な修正案]

### 優先度MEDIUM — 改善推奨
1. [問題点]: [具体的な改善案]
2. [問題点]: [具体的な改善案]

### 競合との差分（追加推奨）
- [不足しているトピック]
- [追加すべきFAQ質問]
- [引用すべきデータ・統計]

### 良い点（維持すること）
- [評価ポイント]
```

診断後、以下を確認する:

「リライトしますか？」
- **全体リライト**: 記事全体をSEO最適化した形に書き直す
- **部分リライト**: 指定したセクション・項目のみ修正する
- **フロントマターのみ修正**: title / description / slug だけ直す
- **修正案の提示のみ**: リライトせず、修正案テキストだけ見せる

---

## ステップ5: リライト実行

ユーザーの選択に応じてリライトを行う。

### 全体リライトの場合

以下を守って書き直す:

- 元の記事の主張・情報・構成の良い点は引き継ぐ
- 診断で指摘した問題点をすべて修正する
- 競合差分で指摘した不足トピックを追加する
- PAA質問に対応したFAQセクションを充実させる（最低3問、推奨5問）
- メインキーワード・共起語の配置を最適化する
- 文字数を競合比較の適正レンジに合わせる
- 冒頭150字以内にメインキーワードを自然に配置する
- 定義文（「〇〇とは〜です」）を入れる
- 引用データには外部リンク（出典URL）を付ける
- H2にメイン/サブキーワードを含める
- H3に疑問形見出しを入れる（強調スニペット・PAA対策）

記事フォーマットは /seo スキルと同一:
```
---
title: [タイトル（35字以内・KW含む）]
description: [メタディスクリプション（120〜160字）]
slug: [url-slug-in-english]
categories: [カテゴリ名]
tags: [タグ1, タグ2, タグ3]
status: publish
---

[本文]
```

### 部分リライトの場合

指定されたセクションのみ修正する。
修正前と修正後を並べて提示し、ユーザーの確認を取ってから反映する。

### フロントマターのみの場合

改善後のフロントマターを提示し、確認後に元ファイルへ反映する。

---

## ステップ6: ファイル保存（リライト実施時）

ユーザー承認後、元ファイルをバックアップしてから上書き保存する:

```bash
cp articles/元ファイル名.md articles/元ファイル名.md.bak
```

保存後、「投稿しますか？」と確認する。
投稿する場合:

```bash
python3 post.py articles/ファイル名.md
```
