# /seo — SEO最適化記事の執筆・投稿

テーマ: $ARGUMENTS

---

WordPress（ai-bpo.site）向けに、SEOチェックリストを満たした記事を執筆する。
ユーザーと対話しながら進めること。各ステップで確認を取り、承認を得てから次へ進む。

---

## ステップ1: テーマ・ターゲットキーワード確認

`$ARGUMENTS` が空または不明確な場合、ユーザーにテーマを確認する。

テーマが決まったら以下を確認する:
- 狙いたいキーワードの候補はあるか（なければステップ2で調査）
- 想定読者は誰か（例: AI初心者、中小企業経営者など）
- 内部リンクしたい既存記事はあるか

---

## ステップ2: 並行調査（3エージェントを同時起動）

**キーワード・SERP調査エージェント**、**競合コンテンツ調査エージェント**、**E-E-A-T素材収集エージェント**を並行起動する。

### キーワード・SERP調査エージェントへの指示

WebSearch を使い以下を調査する:

1. **検索意図の分類**
   - 情報収集型（Informational）/ 比較検討型（Commercial）/ 購買型（Transactional）のどれか
   - ユーザーが本当に知りたいことは何か

2. **SERP分析**
   - 上位表示されているコンテンツの形式（リスト/Q&A/ハウツー/定義型）
   - 強調スニペット（Featured Snippet）の有無と形式
   - People Also Ask（PAA）に出ている関連質問を10個以上抽出
   - 上位ページの文字数・構成パターン

3. **関連キーワード・共起語**
   - メインキーワードの関連語・共起語・LSIキーワード
   - 検索ボリュームが見込める派生キーワード

4. **2026年AI検索への対応**
   - ChatGPT・Perplexity・Geminiでの同テーマ検索結果の傾向
   - AI Overviewに採用されやすいコンテンツ形式の確認

出力: SERP分析レポート（検索意図・推奨フォーマット・PAA質問リスト・推奨文字数）

### 競合コンテンツ調査エージェントへの指示

WebSearch・WebFetch を使い以下を調査する:

1. **上位3〜5記事の構成分析**
   - 各記事の見出し構造（H1〜H3）を抽出
   - カバーしているトピックのリスト化
   - 不足している観点・差別化できる切り口の特定

2. **被リンク・引用されやすい要素の確認**
   - 上位記事が引用しているデータ・統計・調査元
   - 図解・表・オリジナルデータの有無

3. **コンテンツギャップ**
   - 競合が答えていない質問（PAAや関連検索から特定）
   - より詳しく・より正確に書けるポイント

出力: 競合分析レポート（共通構成・差別化ポイント・引用すべきデータ源）

### E-E-A-T素材収集エージェントへの指示

WebSearch を使い以下を調査する（競合エージェントとは独立して調査）:

1. **一次情報・信頼性の高いデータ収集**
   - 政府機関・学術機関・業界団体の統計・調査結果
   - 最新年（2025〜2026年）のデータを優先
   - 引用元URLと数値を明記

2. **専門的な裏付け**
   - 専門家・研究者の見解
   - 公式ドキュメント・ホワイトペーパー

3. **よくある誤解・落とし穴**
   - テーマに関するよくある誤情報
   - 初心者が陥りがちな誤解

出力: E-E-A-T素材リスト（引用可能なデータ・専門情報・要注意ポイント）

## ステップ4: ライターエージェント起動

承認された戦略をもとにライターエージェントを起動する。

### ライターエージェントへの指示

以下のSEOルールをすべて満たした完全な記事を執筆すること。

---

#### SEO必須要件（すべて守ること）

**タイトル（title）**
- メインキーワードを先頭か前半に配置
- 30〜35字以内
- 数字・具体性・読者メリットを含む
- 「最新」「完全」「初心者向け」等の訴求語を適切に使用

**メタディスクリプション（description）**
- 120〜160字
- メインキーワード・サブキーワードを自然に含む
- 記事の価値を明確に伝える
- クリックしたくなる表現にする

**URL スラッグ（slug）**
- 英数字・ハイフン区切り
- メインキーワードをローマ字または英語で表現
- 短く簡潔（3〜5単語）

**見出し構造**
- H1: タイトルと同一（WordPressが自動生成するため記事本文には書かない）
- H2: 主要セクション（メインキーワード・サブキーワードを含む）
- H3: H2の補足・詳細（関連語・PAA質問を見出しに活用）
- H4以下: 必要な場合のみ
- 見出しに疑問形を使うと強調スニペット・PAA対策になる

**本文コンテンツ**
- 冒頭100〜150字以内にメインキーワードを自然に配置
- 共起語・関連キーワードを本文全体に自然に散りばめる
- キーワード密度は1〜2%程度（詰め込まない）
- 推奨文字数を満たす
- 箇条書き・表・数字リストを適切に使用（強調スニペット対策）

**E-E-A-T（経験・専門性・権威性・信頼性）**
- 収集した一次情報・統計データを引用（出典リンク付き）
- 実体験・具体例を含める
- 著者の専門性が伝わる記述を入れる
- 不確かな情報には「〜とされています」「〜と言われています」を使う
- 最新情報（年号・バージョン）を明記する

**強調スニペット・PAA対策**
- 検索意図の核心に対する明快な答えを冒頭付近に配置
- 「〇〇とは、〜です」形式の定義文を入れる
- PAA質問に対応するQ&Aセクションを本文またはFAQセクションに含める
- FAQセクションは最低3問、できれば5問以上

**AI検索（GEO）対策**
- 直接的・明確な答えを提供する（遠回しな表現を避ける）
- 「結論から言うと〜」「端的に言えば〜」の形式を活用
- 情報ソース・根拠を明示する（AI引用時の信頼性向上）
- 構造化された情報（番号リスト・表・見出し）を使う

**画像**
- 各画像のalt属性テキストを `![キーワードを含む説明文](画像URL)` の形式で指定
- ファイル名もキーワードを含む命名を推奨（コメントで指示）
- WebP形式を推奨（コメントで指示）

**内部リンク**
- ユーザーが指定した関連記事へのリンクを本文中の自然な箇所に配置
- アンカーテキストはリンク先のキーワードを含む自然な表現
- 冒頭・中盤・末尾付近に1〜2本ずつ配置

**外部リンク**
- 引用したデータ・統計・一次情報には必ず外部リンクを貼る
- 信頼性の高いドメイン（政府・学術・業界団体）を優先
- `rel="noopener noreferrer"` はWordPressが自動付与するため記述不要

---

## ステップ5: SEOチェック・ファイル保存・投稿

### SEOセルフチェック

記事完成後、以下を確認してから提示する:

**必須項目（必ずチェック）**
- [ ] タイトルにメインキーワードが含まれている
- [ ] タイトルが35字以内である
- [ ] メタディスクリプションが120〜160字である
- [ ] 冒頭150字以内にメインキーワードが含まれている
- [ ] H2見出しに主要キーワードが含まれている
- [ ] FAQセクションが3問以上ある
- [ ] 引用データに外部リンクが付いている
- [ ] 推奨文字数を満たしている
- [ ] 定義文（〇〇とは〜です）が含まれている
- [ ] 内部リンクが含まれている（指定がある場合）

**推奨項目**
- [ ] 共起語・関連キーワードが本文に散りばめられている
- [ ] 番号付きリストまたは箇条書きがある
- [ ] 表が1つ以上ある
- [ ] 画像のalt属性テキストが指定されている
- [ ] slugが英数字・ハイフンのみで3〜5単語

### ファイル保存

チェック完了後、以下のルールで保存する:

- 保存先: `/Users/ando/Dropbox/allsrc/ai-bpo-wp/articles/`
- ファイル名: `YYYY-MM-DD-slug.md`（slugはフロントマターのslugと同一）

### ファクトチェック自動実行

ファイル保存完了後、**ユーザーへの確認なしに** `/fact-check` を自動実行する。

1. `/fact-check articles/YYYY-MM-DD-slug.md` を実行する（引数に保存したファイルパスを渡す）
2. fact-check内の「修正しますか？」確認はスキップし、以下を即座に自動修正する：
   - ❌ 誤り — すべて正確な情報に書き換える
   - ⚠️ 要注意 — すべて更新・表現を修正する
   - ❓ 確認不能 — 「（要出典）」注記 or 削除
3. 修正前に `.bak` バックアップを作成する（`articles/YYYY-MM-DD-slug.md.bak`）
4. 修正完了後、変更内容のサマリーを控えておく（次の「記事提示と確認」で報告）

### 記事提示と確認

記事とSEOチェック結果を提示する。

```
## SEOチェック結果

✅ 必須項目: X/10 クリア
⚠️ 推奨項目: X/5 クリア

[未クリア項目があれば理由と修正案を明示]
```

「いかがでしょうか？修正したい箇所、追加したい情報があれば教えてください。」

修正が確定したら上書き保存する。

### 投稿

ユーザーから投稿の承認を得たら以下を実行する:

```bash
python3 post.py articles/YYYY-MM-DD-slug.md
```

投稿完了後、WordPressの記事URLを報告する。

## 参考: SEO各要素の優先度

| 優先度 | 要素 | 理由 |
|--------|------|------|
| 最高 | タイトル・H1 | 検索エンジンが最も重視 |
| 最高 | 検索意図との一致 | ランキングの根本要件 |
| 高 | メタディスクリプション | CTR（クリック率）に直結 |
| 高 | 冒頭のキーワード配置 | クロール時の重要度評価 |
| 高 | E-E-A-T（一次情報・出典） | 2024〜2026年のGoogleコアアップデート対応 |
| 高 | FAQセクション | 強調スニペット・PAA・AI検索対策 |
| 中 | 内部リンク | サイト全体の権威性向上 |
| 中 | 画像alt属性 | 画像検索・アクセシビリティ |
| 中 | 文字数（適正量） | 競合との比較で判断 |
| 中 | 共起語・関連語 | トピックカバレッジ評価 |
