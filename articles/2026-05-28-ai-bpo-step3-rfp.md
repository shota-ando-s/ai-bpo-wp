[Step 2のスコープ定義](/ai-bpo-step2-scope)で「何をアウトソースするか」が決まりました。次はそれをベンダーに正確に伝えるための「RFP（Request for Proposal：提案依頼書）」を作成します。

RFPの質がそのまま、受け取る提案書の質を決めます。曖昧なRFPを出すと、的外れな提案が来て選定に時間がかかるか、誤解が生まれてトラブルになります。

---

## RFPとは何か

RFPとは、**発注候補のベンダーに対して「こういう業務を委託したい、このような条件で提案してほしい」と伝える公式文書**です。

RFPを作る主な目的は3つです：

1. **自社の要件をベンダーに正確に伝える**（誤解・期待ずれを防ぐ）
2. **複数ベンダーから同じ条件で提案をもらい、比較できるようにする**
3. **選定プロセスの透明性を確保する**（後からの「言った言わない」を防ぐ）

---

## RFPの全体構成

BPO向けRFPの標準的な構成は以下の通りです。

| セクション | 内容 | 重要度 |
|---|---|---|
| ①会社概要 | 自社の事業・規模・業界 | ★★☆ |
| ②RFP発行の目的と背景 | なぜBPOを検討しているか | ★★★ |
| ③委託業務の詳細 | 何をどのように委託したいか | ★★★ |
| ④品質基準・SLA要件 | 最低限のサービス品質 | ★★★ |
| ⑤セキュリティ・機密保持要件 | 情報管理の条件 | ★★★ |
| ⑥価格・契約条件 | 予算感・契約形態の希望 | ★★☆ |
| ⑦提案書のフォーマット指定 | 比較しやすくするための統一様式 | ★★☆ |
| ⑧選定スケジュール | 提案期限・決定時期 | ★★☆ |
| ⑨質問受付方法 | ベンダーからの質問への対応 | ★☆☆ |

## 各セクションの書き方

### ①会社概要

自社の基本情報を簡潔に記載します。業界・事業内容・従業員数・年商規模が最低限必要です。ベンダーが「この規模の企業ならこの体制で対応できる」と判断する材料になります。

```
【会社概要】
・会社名：〇〇株式会社
・業種：製造業（金属部品の設計・製造）
・従業員数：80名
・年商：12億円
・システム環境：Microsoft 365、freee会計、Salesforce
```

### ②RFP発行の目的と背景

なぜBPOを検討しているかを正直に書きます。「人手不足」「コア業務への集中」「スケーラビリティ確保」など、背景を知ることでベンダーは適切な提案ができます。

```
【背景と目的】
当社では経理部門の人員が2名体制で、月末の請求書処理・仕訳入力に
1人あたり月20時間以上を費やしています。中期的な成長に備えて
処理量を増やす必要がある一方、採用難により内部増員が困難な状況です。
BPO活用により、繁閑に対応できる柔軟な処理体制を構築し、
社内経理担当者が月次決算・分析業務に集中できる環境を目指します。
```

### ③委託業務の詳細（最重要）

ここが最も詳しく書くべき箇所です。以下の観点で記載してください。

```
【委託業務の詳細】

■対象業務
・請求書（仕入先からの請求書）の受取・確認・仕訳入力・承認ルーティング

■業務量
・月間処理件数：約200件（繁忙期は最大300件、閑散期は最小150件）
・処理ファイル形式：PDF（メール添付）・紙（郵送→スキャン）
・現在使用システム：freee会計（ベンダーはAPIまたはCSVで入力可）

■現在の業務フロー
①メール受信・郵送受取 → ②内容確認・不備チェック
→ ③仕訳入力（freee） → ④承認者へルーティング
→ ⑤ファイリング（クラウドストレージ）

■委託範囲（案）
・①〜④まで委託（⑤ファイリングは社内で対応）
・不明点・不備は担当者（〇〇）にSlack/メールで連絡

■対象外業務（社内で継続）
・最終承認（経理部長・経営者が承認）
・振込処理
・月次締め・決算対応
```

### ④品質基準・SLA要件

あいまいな品質基準は後でトラブルになります。数値で明記してください。

```
【品質基準・SLA要件（案）】
・入力精度：エラー率0.5%以下（月200件中1件以内）
・処理速度：受取から48時間以内に入力完了
・対応時間：平日9:00〜18:00（質問・不備確認の対応時間）
・月次報告：処理件数・エラー件数・処理時間のレポートを月次で提供
```

### ⑤セキュリティ・機密保持要件

```
【セキュリティ要件】
・ISO 27001取得またはそれに準じたセキュリティ管理体制
・担当者は守秘義務契約の締結
・データは日本国内のサーバーで管理
・第三者への情報提供禁止
・業務終了時のデータ返還・消去の手続き明記
```

### ⑥価格・契約条件

```
【価格・契約条件（希望）】
・契約形態：月額固定料金（処理量に応じた従量制も可）
・予算レンジ：月額5万〜15万円
・契約期間：最低1年（自動更新・3ヶ月前通知で解約可）
・初期費用：可能な限り抑えたい（30万円以内が望ましい）
```

## LLMを使ってRFPを半日で作る手順

RFPはゼロから書くと1〜2週間かかることもあります。LLM（ChatGPT・Claude等）を使えば、草稿を数時間で作成できます。

### ステップ1：情報の整理（30分）

以下の情報をメモ帳やスプレッドシートにまとめます：
- 委託したい業務名と詳細（Step 2の計画書から転記）
- 月間処理量・使用システム
- 品質基準の希望（エラー率・納期等）
- 予算感・契約形態の希望

### ステップ2：LLMにRFP草稿を生成させる（30分）

以下のプロンプトを使ってください：

```
あなたはBPO（ビジネスプロセスアウトソーシング）のRFP（提案依頼書）作成の
専門家です。以下の情報をもとに、BPOベンダー向けのRFPを作成してください。

【会社情報】
・業種：製造業
・従業員数：80名
・使用システム：freee会計、Microsoft 365

【委託したい業務】
・業務名：請求書受取・確認・仕訳入力・承認ルーティング
・月間処理量：約200件
・現在の担当体制：経理2名で合計月40時間

【品質基準の希望】
・エラー率0.5%以下
・受取から48時間以内の入力完了

【予算・契約条件】
・月額5〜15万円
・最低1年契約

上記をもとに、ベンダーへの提案依頼書（A4・3〜5枚程度）を
ビジネス文書として作成してください。
各セクション（目的・委託業務詳細・品質要件・セキュリティ要件・
価格条件・スケジュール・提案フォーマット）を含めてください。
```

### ステップ3：生成された草稿を修正（1〜2時間）

LLMが生成した草稿は必ず確認・修正してください。特に確認すべき点：

- **数値が正確か**（処理量・エラー率・予算等）
- **実際の業務フローと一致しているか**
- **社名・担当者名などの固有情報が正しく入っているか**
- **要件漏れがないか**（次のチェックリストで確認）

### RFP要件漏れチェックリスト

```
□ 委託業務の詳細（フロー・件数・ファイル形式）が書かれているか
□ 使用システム・アクセス方法が明記されているか
□ 品質基準（エラー率・処理速度）が数値で書かれているか
□ セキュリティ要件（ISO取得・データ保管場所等）が書かれているか
□ 価格の形態（固定/従量）と予算感が明記されているか
□ 契約期間と解約条件が書かれているか
□ 選定スケジュール（提案期限・決定時期）が明記されているか
□ 提案書のフォーマット・提出方法が指定されているか
□ 質問受付の方法と期限が書かれているか
```

## RFPでよくある失敗パターン

### 失敗1：業務量・件数が書かれていない

「経理業務の一部」という記述だけでは、ベンダーは見積もりができません。月間件数・処理時間・ファイル形式まで具体的に書くことが必須です。

### 失敗2：品質基準があいまい

「高品質で処理してください」という要件では意味がありません。「エラー率0.5%以下」「48時間以内」のように、測定可能な数値で指定してください。

### 失敗3：予算を書かない

「予算は提案を見てから決める」という姿勢では、ベンダーが提案範囲を見誤ります。「月5〜15万円の予算感」という目安を示すことで、現実的な提案が来ます。

### 失敗4：複数ベンダーに同じRFPを送らない

1社だけに打診すると比較ができず、交渉力もありません。同じRFPを3〜5社に送ることで競争原理が働きます。

### 失敗5：質問受付期間を設けない

ベンダーからの疑問・質問には積極的に答えることが重要です。FAQ形式で質問を集約し、全社に同じ回答を配布すると公平な提案環境が整います。

## RFP配布後のスケジュール管理

RFP配布から選定完了までの標準的なスケジュールは以下の通りです。

| 日程 | イベント |
|---|---|
| Day 0 | RFP配布（3〜5社へ） |
| Day 7 | ベンダーからの質問受付期限 |
| Day 8 | 全社への一斉回答（FAQ配布） |
| Day 21 | 提案書提出期限 |
| Day 22〜28 | 提案書評価・ショートリスト作成 |
| Day 35 | プレゼンテーション（最終候補2〜3社） |
| Day 42〜49 | 最終選定・交渉 |
| Day 56 | 契約締結（Step 5）へ |

## よくある質問（FAQ）

**Q. RFPは必ず正式文書として作らなければいけませんか？**
正式なRFPでなくても、要件を文書化して複数社に同一条件で送ることが重要です。中小企業であれば、2〜3枚のA4資料でも問題ありません。形式より内容の正確さを優先してください。

**Q. どのくらいのベンダーにRFPを送るべきですか？**
3〜5社が適切です。1〜2社では比較と交渉力が弱く、6社以上では選定作業が煩雑になります。あらかじめ[AI BPO比較記事](/ai-bpo-hikaku)などで候補を絞り込んでから送ってください。

**Q. LLMが作ったRFPをそのまま使っても大丈夫ですか？**
そのまま使うのはリスクがあります。LLMは一般的なテンプレートを生成しますが、自社の具体的な数値・フロー・システム情報が正確に反映されているか必ず確認・修正してください。特に品質基準の数値は慎重に設定してください。

**Q. 機密情報をLLMに入力してもいいですか？**
企業の機密情報（売上・顧客データ等）はLLMに入力しないことをお勧めします。業務内容・フロー・処理量など、外部に送るRFPにも記載する情報であれば問題ありません。

**Q. RFPを送ったベンダーが全社から断られた場合はどうすればいいですか？**
要件が厳しすぎる（品質基準・価格・スコープ）か、対応可能なベンダーが少ない業務の可能性があります。他のベンダーへの打診と並行して、Step 2（スコープ定義）に戻って委託範囲の見直しも検討してください。

**Q. RFPは英語で作るべきですか？**
国内ベンダーへの委託であれば日本語で問題ありません。海外ベンダーや外資系BPO会社への依頼の場合は英語版が必要になります。

## まとめ

RFP作成（Step 3）で行うことは3つです：

1. **9セクションの情報を整理する**（特に③委託業務の詳細と④品質基準を丁寧に）
2. **LLMで草稿を生成し、確認・修正する**（半日〜1日で完成）
3. **3〜5社に同一条件で送付し、質問対応する**

良質なRFPが、次の[Step 4（ベンダー評価・選定）](/ai-bpo-step4-vendor-selection)で的確な提案書と比較作業を可能にします。
