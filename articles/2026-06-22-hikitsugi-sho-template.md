退職や異動が決まり、いざ引き継ぎ書を作ろうとすると「何をどこまで書けばいいのか分からない」と手が止まってしまう——これは多くの担当者が抱える共通の悩みです。本記事では、**引き継ぎ書**に盛り込むべき項目一覧、そのままコピーして使えるテンプレート・フォーマット、そして経理業務を例にした具体的な記入例（例文）まで、実務でそのまま使える雛形をまとめて提供します。読み終わるころには、自分の業務に合わせて引き継ぎ書をすぐ書き始められる状態になっているはずです。

> 本記事は [業務引き継ぎ完全ガイド](/gyomu-hikitsugi-guide) のクラスター記事です。

## 引き継ぎ書とは・その目的

引き継ぎ書とは、退職・異動・休職などで担当者が変わる際に、後任者が業務を滞りなく続けられるよう、業務の内容・手順・関係者・注意点を文書化したものです。口頭やメモだけの引き継ぎでは「言った・言わない」が起こりやすく、引き継ぎ漏れがそのまま業務トラブルや顧客クレームにつながります。

引き継ぎ書の目的は、大きく次の3つに整理できます。

- **業務の継続性を守る** — 担当者が抜けても業務が止まらないようにする
- **属人化を解消する** — その人しか知らない「暗黙知」を見える化する
- **後任者の不安を減らす** — 何をいつ誰とやればよいか、判断基準を残す

中小企業では一人が複数業務を兼任しているケースが多く、退職者一人の引き継ぎ不足が経営に直結します。厚生労働省の令和6年雇用動向調査によれば離職率は14.2%で、どの会社でも毎年一定の人が入れ替わります。引き継ぎ書を整える文化があるかどうかが、組織の安定性を大きく左右します。

## 引き継ぎ書に盛り込む項目一覧

まず「何を書くか」を固めることが、引き継ぎ書づくりの8割です。最低限おさえたい項目を表にまとめました。

| 項目 | 内容 | 記入のポイント |
| --- | --- | --- |
| 業務名・概要 | 何の業務か、目的は何か | 後任者が初見でも全体像をつかめる粒度で |
| 業務の頻度・スケジュール | 日次／週次／月次／不定期 | 締め日・期限を明記する |
| 具体的な手順 | 着手から完了までの流れ | 番号付きで時系列に |
| 使用システム・ツール | 名称・ログイン情報の所在 | パスワードは別管理場所を指示 |
| 関係者・連絡先 | 社内外の窓口担当者 | 「誰に聞けば解決するか」を残す |
| 必要書類・ファイル | 保管場所・ファイル名 | 共有フォルダのパスまで書く |
| 注意点・トラブル対応 | よくあるミス・例外処理 | 過去の失敗例こそ価値がある |
| 引き継ぎ状況 | 引き継ぎ済／未完の区分 | 残課題を明確にする |

特に「注意点・トラブル対応」と「関係者・連絡先」は、マニュアルには載っていない暗黙知が詰まる部分で、引き継ぎ書の価値が最も出るところです。チェックリスト形式で抜け漏れを防ぎたい場合は [引き継ぎチェックリスト](/hikitsugi-checklist) も併せて活用してください。

## 引き継ぎ書テンプレート・フォーマット

下記をコピーして、自社の業務に合わせて書き換えるだけで使える雛形です。Word・スプレッドシート・テキストいずれにも貼り付けられます。

```
■ 引き継ぎ書

作成日：YYYY年MM月DD日
前任者：（氏名・部署）
後任者：（氏名・部署）
引き継ぎ完了予定日：YYYY年MM月DD日

────────────────────────
【1】業務名・概要
・業務名：
・目的／ゴール：
・全体の位置づけ：

【2】頻度・スケジュール
・頻度：日次／週次／月次／不定期
・締め日・期限：

【3】業務手順
1.
2.
3.

【4】使用システム・ツール
・ツール名：
・ログイン情報の保管場所：
・操作上の注意：

【5】関係者・連絡先
・社内：（氏名／部署／連絡先）
・社外：（会社名／担当者／連絡先）

【6】必要書類・データの保管場所
・ファイル名：
・保管場所（フォルダパス等）：

【7】注意点・トラブル対応
・よくあるミス：
・例外処理：
・過去のトラブルと対応：

【8】引き継ぎ状況・残課題
・引き継ぎ済み：
・未完／継続対応中：
────────────────────────
```

1業務につき1シート（1枚）を基本とし、担当業務が複数ある場合は業務ごとに作成すると後任者が探しやすくなります。

## 記入例（経理業務の例文）

テンプレートだけでは書き方のイメージがつかみにくいので、「月次の請求書発行業務」を例に実際に記入してみます。そのまま参考にできる例文です。

```
■ 引き継ぎ書

作成日：2026年6月22日
前任者：経理部 山田
後任者：経理部 佐藤
引き継ぎ完了予定日：2026年7月15日

【1】業務名・概要
・業務名：月次請求書の発行・送付業務
・目的：取引先への請求を期日までに正確に行い、入金遅延を防ぐ
・全体の位置づけ：月次締め業務の中核。遅れると入金サイクル全体に影響

【2】頻度・スケジュール
・頻度：月次
・締め日：毎月末日。請求書は翌月5営業日までに送付

【3】業務手順
1. 会計システムで前月の売上データを確定する
2. 請求書テンプレートに取引先別の金額を反映
3. 上長（経理課長）に金額をダブルチェック依頼
4. 承認後、PDF化して取引先へメール送付
5. 送付控えを共有フォルダ「請求書/2026年度」に保存

【4】使用システム・ツール
・ツール名：会計ソフトA、請求書発行サービスB
・ログイン情報：パスワード管理表（金庫内・経理課長保管）
・注意：サービスBは月初にメンテナンスで使えない時間帯あり

【5】関係者・連絡先
・社内：経理課長 鈴木（承認担当）
・社外：取引先C社 経理担当 田中様（請求書様式に独自指定あり）

【6】必要書類・データの保管場所
・ファイル名：請求書テンプレート_最新.xlsx
・保管場所：共有サーバー \経理\請求業務\

【7】注意点・トラブル対応
・よくあるミス：C社のみ請求書に発注番号の記載が必須。漏れると差し戻し
・例外処理：金額に修正が入った場合は必ず赤伝を発行してから再請求

【8】引き継ぎ状況・残課題
・引き継ぎ済み：通常月の発行フロー
・継続対応中：D社との支払サイト変更交渉（7月中に結論予定）
```

このように「過去のミス」「取引先ごとの例外」まで具体的に書くことで、後任者が同じ失敗を繰り返さずに済みます。

## 伝わる引き継ぎ書を書くコツ

同じテンプレートを使っても、書き方しだいで伝わり方は大きく変わります。以下を意識すると後任者が迷いません。

- **専門用語を避け、初見でも分かる言葉で書く** — 社内の略称や担当者しか分からない呼び方には注釈を付ける
- **「なぜそうするのか」を一言添える** — 手順だけでなく理由があると、応用が利き、例外にも対応できる
- **トラブル事例こそ厚く書く** — 順調なときの手順より、つまずいたときの対処が後任者を救う
- **完成を待たず早めに渡す** — 引き継ぎ期間中に一緒に作業しながら追記すると、抜けに気づける

進め方のステップを段取りから知りたい場合は [引き継ぎの進め方](/hikitsugi-susumekata) も参考になります。引き継ぎ書は「一度書いて終わり」ではなく、後任者と一緒にアップデートしていく前提で作るのが、結局いちばん確実です。

## AIで引き継ぎ書・マニュアル作成を支援する

「項目は分かったが、忙しくて書く時間がない」「複数業務分をゼロから作るのは負担が大きい」——これは中小企業で最も多い本音です。

「オフボード」の [AI-BPO導入支援サービス](/hikitsugi-ai-bpo-roadmap) では、AIを活用したマニュアル・引き継ぎ書の自動作成支援を行っています。前任者へのヒアリングや既存資料をもとに、AIがたたき台を生成し、専任スタッフが実務で使える形に整えます。ゼロから書く負担を減らしつつ、属人化した業務を見える化したい企業の引き継ぎを後押しします。

## 引き継ぎ書とマニュアルの違い

混同されがちですが、引き継ぎ書とマニュアルは役割が異なります。

- **引き継ぎ書** — 特定の担当者から後任者へ、「今この瞬間の業務状況」を引き渡すための文書。残課題や進行中の案件など、その時点固有の情報を含む
- **マニュアル** — 担当者が誰であっても通用する、業務の標準的なやり方をまとめた文書。恒久的に使う

引き継ぎ書は「人から人へのバトン」、マニュアルは「組織に残す資産」と考えると分かりやすいでしょう。理想は、引き継ぎを機にマニュアルも整備し、次回以降の引き継ぎを楽にすることです。マニュアルづくりの手順は [引き継ぎマニュアルの作り方](/hikitsugi-manual) で詳しく解説しています。

## よくある質問（FAQ）

### 引き継ぎ書はどのくらいの期間をかけて作ればいいですか？

退職・異動が決まったらすぐ着手するのが理想です。目安として、引き継ぎ期間は2週間〜1か月確保できると、前任者と後任者が一緒に作業しながら追記・確認でき、抜け漏れを防げます。

### 引き継ぎ書はWordとExcelのどちらで作るべきですか？

決まりはありません。手順を文章で説明する業務はWord、項目が多く一覧性が欲しい場合はExcelやスプレッドシートが向きます。本記事のテンプレートはどちらでも使えます。後任者が見やすい形式を優先してください。

### 引き継ぎ書に書く分量はどのくらいが適切ですか？

1業務1枚を基本としつつ、トラブル対応や例外処理は省略しないことが大切です。手順の細部は別マニュアルに切り出し、引き継ぎ書には「残課題」と「注意点」を厚く書くとバランスが取れます。

### 口頭でも引き継ぎするなら引き継ぎ書は不要ですか？

口頭引き継ぎは記憶に残りにくく、後任者が後から見返せません。文書として残すことで、引き継ぎ期間後に疑問が出ても自己解決でき、前任者への問い合わせを減らせます。口頭と文書は補完関係です。

### 引き継ぎ書が間に合わず退職日が来てしまいそうです。どうすれば？

優先順位をつけ、頻度が高く影響の大きい業務から先に書きましょう。最低限「関係者の連絡先」と「進行中の残課題」だけでも残せば、後任者は自走しやすくなります。残りはAI支援サービスの活用も選択肢です。

### 引き継ぎ書のテンプレートは無料で使えますか？

本記事に掲載したテンプレート・フォーマットは自由にコピーしてお使いいただけます。自社の業務に合わせて項目を増減し、雛形として社内で標準化することをおすすめします。

## まとめ

- 引き継ぎ書づくりは「何を書くか」を項目で固めることが8割。表の項目一覧をベースに自社用に調整する
- テンプレート・フォーマットは1業務1枚を基本とし、コピーして使い回せる雛形を社内標準にする
- 記入例のように「過去のミス」「取引先ごとの例外」まで具体的に書くと、後任者の失敗を防げる
- 引き継ぎ書（人から人へのバトン）とマニュアル（組織の資産）を使い分け、引き継ぎを機にマニュアルも整備する
