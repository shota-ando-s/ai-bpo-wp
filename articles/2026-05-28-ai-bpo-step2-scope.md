## スコープ定義で行う3つの判断

1. **コア業務 vs ノンコア業務の分類**（何を社内に残すか）
2. **BPO適性の評価**（アウトソース可能な業務の中で何が向いているか）
3. **優先順位の決定**（どれから着手するか）

## ①コア業務 vs ノンコア業務の分類

### コア業務とは何か

「コア業務」とは、自社の競合優位性・差別化・顧客価値に直結する業務です。競合他社には任せたくない、自社のノウハウや判断が必要な業務です。

**コア業務の例：**
- 商品・サービスの企画・開発
- 顧客との関係構築・営業戦略
- 経営判断・事業戦略立案
- 独自技術・ノウハウを使った製造・加工
- 重要顧客への対応・クレーム処理判断

### ノンコア業務とは何か

競合他社と同様のやり方で行われる、標準化・汎用化された業務です。自社がやっても外部がやっても品質に差が出にくいものです。

**ノンコア業務の例：**
- 請求書・領収書の処理・仕訳入力
- 給与計算
- 一般的な問い合わせへの一次回答
- データ入力・集計・帳票作成
- 定型レポートの作成・送付
- 採用サポート（求人票作成・応募者スクリーニング）

### 分類の判断チェックリスト

以下の問いに「はい」が多い業務ほど、アウトソース適性が高いです。

```
□ この業務を競合他社が行っても、顧客価値に違いは出ないか
□ 外部の専門家（BPO業者）でも同等品質で行えるか
□ マニュアルに落とせるルール・判断基準があるか
□ 機密・個人情報のリスクを適切に管理できるか
□ この業務を社内でやる理由が「習慣・慣れ」だけではないか
```

## ②BPO適性の5軸評価フレームワーク

Step 1で計測した業務データをもとに、以下の5軸でBPO適性を定量評価します。

### 5軸評価マトリクス

| 評価軸 | 1点 | 3点 | 5点 |
|---|---|---|---|
| **ルール化可能性** | 判断が複雑で属人的 | 一部ルール化可能 | 完全にマニュアル化できる |
| **反復性** | 毎回内容が変わる | 月に数回程度繰り返す | 毎日・毎週同じ作業 |
| **コア業務でないか** | 競合優位性に直結 | やや関連する | 全く関係ない汎用業務 |
| **データ化可能性** | 紙・口頭・感覚的判断 | 一部デジタル化されている | 完全デジタル、ログ取得可能 |
| **削減効果の大きさ** | 月10時間未満 | 月10〜30時間 | 月30時間以上 |

**合計25点満点で判定：**
- 20〜25点：BPO最優先候補（まず着手）
- 14〜19点：条件次第でBPO可能（第二フェーズ）
- 8〜13点：部分的にBPOを検討
- 5〜7点：社内維持を推奨

## ③ROI試算で最終判断する

適性スコアが高くても、コストが合わなければBPOする意味がありません。必ずROI試算を行ってください。

### ROI試算の計算式

```
年間削減コスト = 現状の年間人件費（工数×単価×12ヶ月）

年間BPO委託費 = ベンダー見積もり × 12ヶ月

投資コスト = 初期導入費用 + 移行準備コスト（内部工数）

ROI = (年間削減コスト − 年間BPO委託費) / 投資コスト × 100（%）

回収期間 = 投資コスト / 月間純削減コスト
```

### ROI試算の具体例

**ケース：請求書処理・仕訳入力をBPOに委託**

| 項目 | 金額 |
|---|---|
| 現状人件費（月40h × 3,500円/h × 12ヶ月） | 168万円/年 |
| BPO委託費（月8万円 × 12ヶ月） | 96万円/年 |
| 年間純削減コスト | **72万円/年** |
| 初期導入費（契約・移行準備） | 50万円 |
| 回収期間 | **約8ヶ月** |
| 3年ROI | **約332%** |

ROIがプラスかつ回収期間が24ヶ月以内であれば、BPO導入の経済的合理性があると判断できます。

## 優先度マトリクス：どれから始めるか

スコアとROIが算出できたら、以下のマトリクスで着手順序を決めます。

```
         ROI（高）
            │
第2着手    │  第1着手（最優先）
 ───────────┼───────────
第4着手    │  第3着手
            │
         ROI（低）
      BPO適性（低）───────BPO適性（高）
```

**第1着手（最優先）：** BPO適性スコア高 × ROI高
→ まずここから始める。成功体験を積む。

**第2着手：** BPO適性スコア低 × ROI高
→ 技術的難易度は高いが効果が大きい。第2フェーズで取り組む。

**第3着手：** BPO適性スコア高 × ROI低
→ 楽に委託できるが効果は小さい。余裕ができたら。

**第4着手（後回し）：** BPO適性スコア低 × ROI低
→ 社内維持が現実的。BPO化は後回しまたは不要。

## AIを使ったスコープ定義の効率化

### LLMによる自動スコアリング

Step 1で作成した業務インベントリをLLMに入力し、5軸評価を自動化できます。

**プロンプト例：**
```
以下の業務リストについて、BPO適性を5軸（ルール化可能性・反復性・
コア業務でないか・データ化可能性・削減効果）でそれぞれ1〜5点評価し、
合計スコアと優先順位の推奨をJSON形式で出力してください。

業務リスト：
- 請求書処理（月40h、経理2名担当、freee使用）
- 給与計算（月20h、人事1名担当、SmartHR使用）
- 一次問い合わせ対応（月60h、CS3名担当、メール・電話）
[以下続く]
```

このプロンプト1回で、20〜30業務の一次スコアリングが数分で完了します。人間の最終確認は必要ですが、棚卸しと評価作業全体を大幅に短縮できます。

### ROI自動試算ツール

Excelベースのシンプルな試算ツールをLLMに生成してもらうことも可能です。

```
Excelで使えるROI試算テンプレートを作成してください。
入力：業務名・月工数・時間単価・BPO月額費用・初期費用
出力：年間削減コスト・年間BPO費・純削減コスト・回収期間・3年ROI
```

## アウトソースしてはいけない業務リスト

BPO適性が低く、原則として外部委託すべきでない業務を確認しておきます。

### 絶対に委託してはいけない業務

- **経営判断・意思決定**（経営会議、M&A判断等）
- **コア技術の開発・改良**（独自製品・サービスの設計）
- **重要顧客との関係管理**（VIP顧客への直接対応）
- **法的責任が伴う最終判断**（契約締結、コンプライアンス判断）

### 委託可能だが慎重に検討すべき業務

- **個人情報を大量に扱う業務**（情報漏洩リスクの管理が必要）
- **クレーム対応の最終回答**（一次受付はBPO可、最終判断は社内）
- **採用最終選考・面接**（カルチャーフィットの判断は社内に残す）
- **経理の最終承認・締め業務**（日常処理はBPO可、決算締めは慎重に）

## よくある質問（FAQ）

**Q. 「とりあえず全部アウトソース」はなぜダメなのですか？**
全業務を一度に委託すると、トラブル発生時の原因特定が困難になり、ベンダーへの依存度が高まります。また、移行準備コストと管理コストが一気に増大します。「小さく始めて成功体験を積む」アプローチの方が、結果として早く・確実に展開できます。

**Q. BPO適性スコアの評価は誰がやるべきですか？**
現場担当者と経営者（または管理職）の両者が評価し、すり合わせることをお勧めします。現場担当者は実務の複雑さをよく知っており、経営者は戦略的重要性を判断できます。どちらか一方の評価だけでは偏ります。

**Q. ROIがマイナスになる業務はBPOしてはいけないですか？**
コスト面ではマイナスでも、「人材をコア業務に集中させる」「採用難で人手が確保できない」「品質安定化」などの非財務メリットで判断するケースもあります。ただし、非財務メリットは定量化が難しいため、慎重に判断してください。

**Q. 適性スコアが似ている業務が複数ある場合、どれを先にBPOすべきですか？**
「最も工数が大きい業務」「担当者が退職リスクを抱えている業務」「繁忙期に最も負荷がかかる業務」の3点を優先基準にしてください。

**Q. 部分的なBPO（業務の一部だけ委託）は可能ですか？**
可能です。たとえば「経理業務全体」ではなく「請求書の入力部分だけ」というスコープ設定もできます。ただし分業が複雑になるほど引き継ぎコストと管理コストが増えるため、可能な限り「一連の業務フロー」としてまとめて委託することをお勧めします。

**Q. スコープ定義後に委託範囲を変更できますか？**
契約後の範囲変更は追加費用が発生することが多いため、Step 2の時点でしっかり検討してください。ただし、パイロット運用（Step 7）後の調整は一般的に認められています。契約書にスコープ変更の条件を明記しておくことが重要です。

**Q. IT導入補助金はスコープ定義の段階から申請できますか？**
IT導入補助金はBPOサービス（ソフトウェア・クラウドツール）への補助であり、コンサル費や委託業務費は対象外です。スコープ定義にツールを使う場合、そのツール費用が補助対象になる可能性があります。詳しくは[IT導入補助金の解説記事](/it-hojo-ai-bpo)を参照してください。
