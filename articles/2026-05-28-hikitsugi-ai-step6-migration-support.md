契約が締結されたら、次はベンダーへの「引き継ぎ」です。この段階が、BPO導入の成否を決めると言っても過言ではありません。

「うちの業務は複雑で、文書化できない」と言う企業ほど、引き継ぎが失敗します。その「複雑さ」を整理して文書化するのが、このStep 6の目的です。ヒキツギAIは業務マニュアルの作成から引き継ぎ計画の策定・ベンダースタッフへのトレーニングまでをサポートします。

---

## 引き継ぎ失敗の典型パターン

### パターン1｜「やってみればわかる」で引き継いだ

担当者が「口頭で説明すれば大丈夫」と思って進めた結果、ベンダースタッフが理解しきれず、品質が低下。「こんな説明では無理」となって関係が悪化するケースがあります。

### パターン2｜マニュアルが「作業手順書」止まり

「何をするか」は書いてあるが「なぜそうするか」「例外の場合どうするか」がない。ルーティン作業は問題なくこなせるが、イレギュラーな状況になるたびに問い合わせが来る、という状態になります。

### パターン3｜移行期間が短すぎた

「早く切り替えたい」という焦りから引き継ぎ期間を短く設定した結果、移行直後に大量のエラーが発生。それを修正する工数が発生し、かえって現場の負荷が増えるケースがあります。

## ヒキツギAIのStep 6支援 全体の流れ

### フェーズ1｜業務フロー・マニュアルの作成（2〜4週間）

現場担当者へのヒアリングをもとに、業務マニュアルセットを作成します。

**私たちが作成するマニュアルセット：**

**①業務フロー図（全体像）**

業務の全体の流れを視覚化します。どこから情報が入ってきて、どこへ出ていくのか、誰がどの段階で関与するのかを一目で分かる形にします。

```
【業務フロー図の要素】
入力 → 処理ステップ1 → 判断分岐 → 処理ステップ2 → 出力
                              ↓
                           例外処理フロー
```

**②作業手順書（ステップバイステップ）**

各作業の「手順」を、初めて行う人でも理解できるレベルで記述します。

```
【作業手順書の構成】
1. 業務の目的・完了基準
2. 前提条件（○○が完了していること等）
3. 手順（番号付き箇条書き）
4. 各手順のスクリーンショット・画像
5. よくある疑問（FAQ）
6. 注意事項・やってはいけないこと
```

**③判断フローチャート（例外対応）**

「○○の場合はどうするか」という判断が必要な場面のフローを設計します。これがあることで、ベンダースタッフが自律的に判断できる範囲が広がり、問い合わせ件数が減ります。

```
例：請求書処理の例外判断
「請求書の金額が前回と10%以上異なる場合」
  → 担当者に確認メール送付
  → 承認が得られた場合 → 通常処理
  → 48時間以内に返答なし → エスカレーション先Aへ連絡
```

**④チェックリスト（品質管理用）**

業務完了時に確認すべき項目をチェックリスト形式でまとめます。ベンダーの品質チェックとしても、貴社のモニタリングとしても使えます。

**⑤用語集・FAQ集**

業界特有の用語・社内特有の用語を辞書化します。「この書類は何と呼ぶのか」「〇〇部品番号の形式は？」といった暗黙知を明示します。

---

### フェーズ2｜移行計画の策定（1週間）

マニュアルと並行して、「いつ・何を・どの順番で移行するか」の計画を策定します。

**移行計画に含める内容：**

```
【移行スケジュール（例）】
W1〜W2：マニュアル作成・ベンダーへの共有
W3：ベンダースタッフへの説明会（トレーニング）
W4：シャドーイング期間（ベンダーが見学・質問する期間）
W5〜W6：並行運用期間（社内とベンダーが同時に処理）
W7：ベンダーへの完全移管（社内担当者はレビューのみ）
W8以降：パイロット運用開始（Step 7へ）
```

**並行運用（パラレルラン）の設計：**

いきなり完全移管するのではなく、一定期間は「社内も同じ業務を続けながらベンダーにもやってもらい、結果を比較する」並行運用を設けることを推奨します。

これによって：
- ベンダーの処理結果を正解と照合できる
- 問題を小さい規模のうちに発見できる
- 担当者の不安を和らげながら移行できる

---

### フェーズ3｜ベンダースタッフへのトレーニング（1〜2週間）

マニュアルを渡すだけでは不十分です。私たちはベンダースタッフへのトレーニングセッションを企画・実施します。

**トレーニングプログラムの構成：**

| セッション | 内容 | 時間 |
|---|---|---|
| オリエンテーション | 貴社の事業概要・業務の背景・目的 | 30分 |
| 業務全体説明 | フロー全体の解説 | 60分 |
| 各作業の手順説明 | 手順書を使った詳細説明 | 120分 |
| 例外処理の演習 | ケーススタディ形式 | 60分 |
| Q&Aセッション | 疑問点の解消 | 30〜60分 |

トレーニング後には理解度確認テスト（筆記）を実施し、理解が不十分な項目を特定して追加説明を行います。

**その後のシャドーイング期間：**
ベンダースタッフが貴社担当者の作業を「見学・補助する」期間を設けます。実際の業務を通じて学ぶことで、マニュアルだけでは伝わらない「コツ」や「現場感」を習得してもらいます。

---

### フェーズ4｜社内スタッフへの「業務移管後」の準備支援

社内担当者への配慮も必要です。「自分の仕事がなくなる」という不安、または「楽になったが何をすべきか」という状態になる可能性があります。

**私たちが支援すること：**
- 移管後に社内担当者が担う「管理・モニタリング業務」の設計
- ベンダーとの窓口担当者（社内BPOマネージャー）の役割定義
- 社内スタッフへの移管目的・背景の説明資料作成（経営者からの発信をサポート）

## ヒキツギAIが特に重視するポイント

### 「なぜそうするのか」を必ず書く

手順書の「なぜ」が抜けていると、状況が変わったときに対応できません。例えば「請求書は必ず翌日処理すること」という手順に対して「なぜ翌日なのか」（入金確認のタイミングと連動しているため）を記載しておけば、繁忙期に処理が遅れた際の優先度判断ができます。

### マニュアルは「生き物」として管理する

一度作ったマニュアルを更新しないでいると、実態と乖離が生じます。私たちは「マニュアル更新ルール」（更新のタイミング・担当者・承認フロー）を合わせて設計し、長期的に使えるドキュメント管理の仕組みを構築します。

## よくある質問

**Q：既に業務マニュアルがある場合、ゼロから作り直す必要がありますか？**
A：既存のマニュアルをベースに、不足項目の補完・最新化・BPO対応版への改訂を行います。ゼロからの場合より費用・期間を短縮できます。

**Q：マニュアルは日本語だけでいいですか？オフショアベンダーを使う場合は？**
A：オフショア（海外）ベンダーを使う場合は英語版の作成も対応できます。別途費用が発生しますが、翻訳支援も行います。

**Q：並行運用期間はどのくらい設けるべきですか？**
A：標準は2〜4週間です。業務の複雑さとベンダーの習熟度によって変わります。品質が確認できるまで延長することを推奨します。
