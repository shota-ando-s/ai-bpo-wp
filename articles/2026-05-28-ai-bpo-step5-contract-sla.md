## BPO契約書の構成

標準的なBPO契約書は以下の7つのパートで構成されます。

| パート | 内容 | 重要度 |
|---|---|---|
| ①委託業務の定義 | 何を委託するか（スコープ） | ★★★ |
| ②SLA（サービスレベル合意） | 品質・速度・可用性の基準 | ★★★ |
| ③機密保持・情報セキュリティ | データの取り扱いルール | ★★★ |
| ④知的財産権 | 業務成果物の帰属 | ★★☆ |
| ⑤料金・支払条件 | 費用・支払いサイト・値上げルール | ★★☆ |
| ⑥免責事項・損害賠償 | トラブル時の責任範囲 | ★★★ |
| ⑦契約期間・解約条件 | いつまで・どう終わらせるか | ★★★ |

---

## ①委託業務の定義（スコープ）

ここが曖昧だと「言った言わない」のトラブルが多発します。

**明記すべき内容：**
- 委託する業務の名称と詳細な説明
- 委託しない業務の範囲（除外事項）
- 業務量（月間処理件数・受付時間等）
- 使用するシステム・ツール
- 成果物の形式（CSV・PDF等）と納品方法

**注意ポイント：**
スコープクリープ（委託範囲が少しずつ拡大する現象）を防ぐため、「本契約に定める業務以外の依頼は、別途見積もりの上、書面で合意する」という一文を必ず入れてください。

## ②SLA（サービスレベル合意）の設計

SLAはBPO契約の核心部分です。品質・速度・可用性を数値で合意します。

### SLAに定める主要指標

| 指標 | 内容 | 数値例 |
|---|---|---|
| **処理精度（エラー率）** | 誤入力・見落としの割合 | 0.5%以下 |
| **処理速度（ターンアラウンド）** | 受付〜完了までの時間 | 48時間以内 |
| **稼働率（可用性）** | システム・体制の稼働保証 | 99%以上（平日） |
| **応答時間** | 質問・問い合わせへの返答時間 | 4時間以内（営業時間内） |
| **インシデント対応時間** | セキュリティ問題等の初動 | 1時間以内に報告 |

### SLA数値の設定方法

SLAの数値を決める際の考え方：

**①現状値を基準にする**
現在社内で処理している際の品質・スピードを把握し、それを最低ラインとして設定します。「今の担当者がやる場合のエラー率が1%なら、BPOに0.5%を求める」という考え方です。

**②業界標準を参考にする**
BPO業界の一般的なSLA：
- 経理・データ入力系：エラー率0.3〜0.5%、処理速度24〜48時間
- コールセンター系：応答率90%以上、一次解決率80%以上
- 人事・採用系：応募スクリーニング48時間以内

**③ペナルティと現実性のバランスをとる**
厳しすぎるSLAを設定すると、ベンダーが達成不可能として辞退するか、ペナルティ回避のために不正な報告をする可能性があります。「少し厳しいが達成可能」な水準が適切です。

### SLA違反時のペナルティ設計

SLAには必ず「違反時の対応」を明記してください。

```
【SLAペナルティの例】
・エラー率が0.5%を超えた月：当月料金の5%クレジット
・処理速度SLAを3回連続違反：契約解除権の発生
・セキュリティインシデント発生：原状回復義務＋損害賠償

【SLAレビューの仕組み】
・月次レポート提出義務（処理件数・エラー件数・処理時間）
・四半期ごとのSLAレビュー会議
・年次のSLA見直し交渉権
```

## ③機密保持・情報セキュリティ

### 機密保持協定（NDA）の必須条項

- 機密情報の定義（何が機密か）
- 目的外使用の禁止
- 第三者への開示禁止（サブコントラクターへの制限含む）
- 契約終了後の情報返還・消去義務
- 違反時の損害賠償

### 情報セキュリティの具体的要件

```
【情報セキュリティ要件の例】
・情報管理体制：ISO 27001取得済みの管理体制の維持
・アクセス管理：最小権限の原則（必要な担当者のみアクセス）
・通信の暗号化：TLS 1.2以上での通信
・データ保管：日本国内のサーバーで保管
・持込デバイス：私用デバイスの業務使用禁止
・インシデント報告：発見から1時間以内の第一報
・定期監査：年1回のセキュリティ監査報告書の提出
```

## ④知的財産権

BPO業務で生成される成果物（報告書・データ・プログラム等）の著作権・所有権が誰に帰属するかを明確にしてください。

**原則：** 委託者（自社）に帰属するよう契約書に明記する。

特にAIを活用した業務では「AIが生成したコンテンツの権利」が曖昧になりがちです。「本業務の過程で生成された成果物の一切の知的財産権は委託者に帰属する」と明記しましょう。

## ⑤料金・支払条件

**明記すべき項目：**
- 月額費用の計算方法（固定か従量か、計算基準）
- 支払いサイト（月末締め翌月末払い等）
- 価格改定のルール（年に1回まで・事前3ヶ月通知等）
- 追加業務の単価と見積もり手続き
- 消費税の扱い

**注意：** 「市場動向に合わせて価格を改定できる」という曖昧な条項を入れさせないことが重要です。「値上げ率は年CPI（消費者物価指数）の変動率以内」等、数値で縛ることを求めてください。

---

## ⑥免責事項・損害賠償

トラブル発生時に誰がどこまで責任を負うかを明確にします。

**ベンダー側が設定しがちな免責事項（注意が必要）：**
- 「自然災害・通信障害等の不可抗力は免責」→ 合理的だが範囲を明確に
- 「賠償額は当月委託料を上限とする」→ 損害が大きい場合は不利
- 「間接損害は賠償しない」→ 業績影響が出た場合に問題になる

**交渉で求めるべき修正：**
- 賠償上限を「過去3〜6ヶ月の委託料の合計」に引き上げ
- セキュリティインシデントは上限なし（または別途上限設定）
- 故意・重過失の場合は上限の適用外

---

## ⑦契約期間・解約条件

**標準的な設定：**
- 契約期間：1年（自動更新）
- 解約通知：3〜6ヶ月前の書面通知
- 即時解約事由：重大な契約違反・破産・セキュリティインシデント

**交渉で確保すべき権利：**
- パイロット期間中の無条件解除権
- SLA未達成が継続する場合の解除権
- 担当者が著しく不適切な場合の変更要求権

## AIを使った契約書レビュー

### AI契約書レビューツールの活用

弁護士に依頼する前の一次チェックとして、AI契約書レビューツールが有効です。

**主要ツール：**

| ツール | 特徴 | 費用目安 |
|---|---|---|
| LegalForce | 国内シェアNo.1、リスク条項を色分け表示 | 月額5万円〜 |
| GVA contract | SME向けコスパ良好、AI抽出精度が高い | 月額3万円〜 |
| CLM（Contract Lifecycle Management） | 欧米系、英語対応が強い | 月額3万円〜 |
| LLM（ChatGPT等）直接活用 | 低コスト、精度は上記より落ちる | 月額数千円〜 |

### LLMで契約書を一次チェックする方法

```
以下のBPO契約書の条項を確認し、委託者（甲）にとって
不利または曖昧な条項を指摘してください。
特に以下の観点でチェックしてください：
①SLAの数値コミットが不十分でないか
②賠償上限が著しく低くないか
③情報セキュリティ要件が具体的か
④解約条件が委託者に不利でないか
⑤スコープ変更・価格改定の条件が曖昧でないか

---
[契約書の条文テキストを貼り付け]
```

**活用の注意点：**
- LLMによるチェックはあくまで一次確認
- 重要・高額契約は必ず弁護士（特にBPO・IT分野）の最終確認を受けること
- 機密情報を含む契約書はセキュアな環境で処理すること

## 契約書に追記すべき「AI BPO特有の条項」

従来のBPO契約書に加えて、AI活用業務では以下の条項も必要です。

**AI利用の透明性：**
```
受託者（乙）は、本業務においてAI・機械学習ツールを使用する場合、
その旨を委託者（甲）に事前に通知し、承認を得るものとする。
また、使用するAIツールの名称・バージョン・学習データの概要を
開示する義務を負う。
```

**AIの生成物の品質責任：**
```
AI・自動化ツールによる処理結果に誤りが生じた場合も、
受託者（乙）が最終的な品質責任を負うものとし、
「AIの判断によるもの」を免責の理由とすることはできない。
```

**データ学習への使用禁止：**
```
受託者（乙）は、委託者（甲）のデータを自社または第三者の
AIモデルの学習データとして使用してはならない。
```

## 契約締結の最終チェックリスト

```
【委託業務の定義】
□ 委託業務のスコープが具体的に明記されているか
□ スコープ外業務の取り扱いルールがあるか

【SLA】
□ エラー率・処理速度等が数値で明記されているか
□ SLA違反時のペナルティが設定されているか
□ 月次レポートの提出義務があるか

【セキュリティ・機密保持】
□ 機密情報の定義が明確か
□ 情報の返還・消去義務があるか
□ インシデント報告義務の時間が設定されているか

【料金・契約期間】
□ 価格改定のルールが数値で縛られているか
□ 解約通知期間が明記されているか
□ パイロット期間の特約があるか

【責任・免責】
□ 賠償上限が委託金額の何ヶ月分かが明記されているか
□ 重大違反時の即時解除条項があるか
```

## よくある質問（FAQ）

**Q. 弁護士に依頼しなくても契約書は作れますか？**
定型的な少額委託であれば、ベンダーが提示する標準契約書＋AIレビューでも対応可能です。ただし、月額10万円超・個人情報を扱う・長期契約（2年以上）の場合は、弁護士の確認を強くお勧めします。費用は5〜15万円程度です。

**Q. ベンダーが契約書の修正を拒否した場合はどうすればいいですか？**
「修正しない理由」を明確に聞いてください。合理的な理由がある場合（業界標準の免責条項等）は受け入れ可能ですが、「これが弊社の標準で変更はできない」という回答だけの場合は、そのベンダーのリスクを十分理解した上で判断してください。重要条項の修正を拒否するベンダーは、トラブル発生時にも誠実に対応しない可能性があります。

**Q. SLAの数値は高く設定するほどいいですか？**
高すぎる数値は逆効果です。達成不可能なSLAを設定すると、ベンダーが条件を受け入れた後、実態として達成できない状況が続きます。報告書の数値を調整する（正確でない報告をする）リスクもあります。「現実的に達成可能かつ自社の業務要件を満たす」水準が最適です。

**Q. 個人情報保護法上の注意点はありますか？**
BPOは個人情報の「委託」に当たります。個人情報保護法上、委託者は委託先を「適切に監督する義務」があります。契約書でのセキュリティ要件の明記、定期的な監査（または報告受領）、サブコントラクターへの流用禁止条項が必要です。

**Q. 契約書はどのツールで作成・管理すればいいですか？**
DocuSign、クラウドサイン等の電子契約サービスで締結・管理することをお勧めします。紙の契約書は紛失リスクがあり、内容の検索・参照が困難です。電子契約なら自動アラート（更新時期・解約通知期限等）も設定できます。

## まとめ

BPO契約・SLA設計（Step 5）で確認すべきことは3つです：

1. **7パートの契約書を全て確認し、曖昧な条項を数値・具体的な記述に修正する**
2. **SLAは現実的かつ測定可能な数値で設定し、ペナルティも明記する**
3. **AI契約書レビューツールで一次チェックし、重要契約は弁護士に最終確認してもらう**
