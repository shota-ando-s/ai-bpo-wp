担当者が退職した後になって「あの取引先の対応、誰も引き継いでいなかった」「パスワードがわからず業務が止まった」——こうした引き継ぎ漏れは、中小企業ほど深刻なダメージになります。属人化した業務を口頭だけで引き継ぐと、必ずどこかに穴が空きます。そこで役立つのが、項目を一つずつ潰していく**業務引き継ぎチェックリスト**です。本記事では、引き継ぐ側・受け取る側それぞれが使える項目一覧を表で提供し、抜け漏れを防ぐ運用のコツまで解説します。

> 本記事は [業務引き継ぎ完全ガイド](/gyomu-hikitsugi-guide) のクラスター記事です。

## なぜ業務引き継ぎにチェックリストが必要なのか

引き継ぎは「やったつもり」になりやすい作業です。本人は伝えた気でいても、受け取る側は何を聞いていないかすら把握できません。記憶や善意に頼った引き継ぎは、対応の優先順位が曖昧になり、重要度の高い業務ほど後回しになりがちです。

この問題は経営リスクに直結します。厚生労働省の調査によると、令和5年の離職率は14.2%にのぼります。およそ7人に1人が1年間で職場を離れる計算であり、引き継ぎは「いつか起きること」ではなく「毎年必ず起きること」です。

さらに、帝国データバンクの調査では、人手不足や後継者・従業員の退職が引き金となった「退職型」の倒産は2024年に87件と過去最多を記録しました（[出典：帝国データバンク](https://www.tdb.co.jp/report/economic/20250309_taisyoku/)）。人が抜けることそのものが企業の存続を脅かす時代に、引き継ぎの質を個人任せにするのは危険です。

チェックリストには次の効果があります。

- **抜け漏れの可視化**：何が終わって何が残っているか一目でわかる
- **属人化の解消**：頭の中の暗黙知を項目として外に出せる
- **進捗管理**：管理者が引き継ぎ状況を客観的に把握できる
- **品質の標準化**：誰が引き継いでも一定水準を確保できる

## 【引き継ぐ側】業務引き継ぎチェックリスト

退職・異動する本人が、引き継ぎ開始前から最終日までに確認すべき項目です。完了欄にチェックを入れながら進めてください。

| 項目 | 内容 | 完了 |
|------|------|------|
| 担当業務の棚卸し | 日次・週次・月次・不定期の業務をすべて書き出す | ☐ |
| 業務マニュアル | 手順書・作業フローを最新化、または新規作成 | ☐ |
| 取引先・関係者リスト | 担当者名・連絡先・取引条件・注意点を整理 | ☐ |
| 進行中の案件 | 現在のステータスと次のアクションを明記 | ☐ |
| 定例スケジュール | 締め日・提出期限・定例会議などを一覧化 | ☐ |
| アカウント・パスワード | 使用システムのID・権限を引き継ぎ（安全な方法で） | ☐ |
| 保管場所 | 書類・データ・備品の保管場所を明示 | ☐ |
| 過去のトラブル事例 | 起きやすい問題と対処法の共有 | ☐ |
| 暗黙のルール | 社内の慣習・キーパーソン・根回しの順序 | ☐ |
| 引き継ぎ書 | 上記をまとめた文書を作成し受領サインを得る | ☐ |

特に漏れやすいのが「暗黙のルール」と「過去のトラブル事例」です。マニュアルに書きにくい情報こそ、口頭ではなく文書として残すよう意識してください。書き方の具体例は [引き継ぎ書の書き方](/hikitsugi-sho-template) で詳しく解説しています。

## 【受け取る側・管理者】業務引き継ぎチェックリスト

引き継ぎは「受け取って終わり」ではありません。受け取る側と管理者がチェックすることで、漏れを二重に防げます。

| 項目 | 内容 | 完了 |
|------|------|------|
| 引き継ぎ書の受領 | 内容を読み込み、不明点を質問する | ☐ |
| 実務の試行 | 本人がいるうちに自分で一度やってみる | ☐ |
| 取引先への挨拶 | 後任として顔合わせ・連絡先交換を済ませる | ☐ |
| アカウント移管 | 権限の付与・パスワード変更を確認 | ☐ |
| 不明点の洗い出し | 「聞いていないこと」をリスト化して再確認 | ☐ |
| 緊急時の連絡先 | トラブル時に誰へ相談するかを把握 | ☐ |
| 引き継ぎ完了の承認 | 管理者が内容を確認し正式に完了とする | ☐ |
| 退職後の質問窓口 | 一定期間は元担当へ連絡可能か取り決める | ☐ |

管理者の役割は、引き継ぐ側と受け取る側の認識のズレを埋めることです。両者だけに任せると「言った・聞いていない」のトラブルが起きやすいため、第三者として完了を承認するプロセスを必ず設けましょう。退職にともなう手続き全体の漏れを防ぐには [退職手続きチェックリスト](/offboarding-checklist) も併せて確認してください。

## 退職時と異動時でチェック項目はどう違うか

引き継ぎと一口に言っても、退職と異動では押さえるべきポイントが異なります。

**退職時**は、本人が社外へ出るため「後から聞き直せない」前提で進める必要があります。アカウントの完全な移管・削除、取引先への正式な後任紹介、退職後の質問対応の可否など、関係を断つことを見据えた項目が増えます。情報漏えいやアカウントの放置はセキュリティ事故につながるため、特に慎重さが求められます。

**異動時**は、本人が社内に残るため緊急時に確認できる余地があります。ただし「いつでも聞ける」と考えて引き継ぎを簡略化すると、異動先の業務に追われて結局フォローできなくなります。異動特有の論点（部署間の連携、引き継ぎ期間の設定など）は [人事異動の引き継ぎ](/idou-hikitsugi) で詳しく整理しています。

どちらの場合も、引き継ぎに必要な期間を逆算して計画を立てることが重要です。適切な期間の目安は [引き継ぎ期間の目安](/hikitsugi-kikan) を参考にしてください。

## 抜け漏れを防ぐ運用のコツ

チェックリストは作っただけでは機能しません。運用の質が結果を左右します。

1. **早めに着手する**：退職・異動が決まったらすぐ始める。最終日直前のまとめ作業では必ず漏れます。
2. **業務の棚卸しから始める**：いきなり引き継ぎ書を書かず、まず全業務を書き出す。リスト化されていない業務は引き継げません。
3. **受け取る側に実際にやらせる**：説明を聞くだけでなく、本人がいるうちに試行してもらう。これが最大の漏れ防止策です。
4. **完了の定義を決める**：「引き継ぎ済み」の基準を管理者が明確にする。曖昧なままだと未完了のまま退職日を迎えます。
5. **進捗を見える化する**：完了欄を共有し、誰でも進捗を確認できるようにする。

これらを毎回ゼロから手作業で行うと、担当者の負担が大きく形骸化しがちです。引き継ぎを仕組みとして回す体制づくりが、持続的な抜け漏れ防止につながります。

## チェックリストと引き継ぎ書はセットで使う

チェックリストは「やるべきことの一覧」、引き継ぎ書は「業務の中身を記録した文書」です。役割が異なるため、両者を併用してはじめて引き継ぎが完成します。

チェックリストで全体の進捗を管理しながら、各項目の詳細を引き継ぎ書に落とし込んでいくのが基本の流れです。チェックリストだけでは具体的な手順が伝わらず、引き継ぎ書だけでは何が残っているか把握できません。

引き継ぎ書をどう構成し、何を書けばよいかは [引き継ぎ書の書き方](/hikitsugi-sho-template) にテンプレートと記入例を用意しています。チェックリストの各項目と対応させながら作成すると、抜け漏れのない引き継ぎ文書が完成します。

## 引き継ぎの属人化を根本から解決する「オフボード」

チェックリストや引き継ぎ書を整えても、結局は担当者の手作業に依存します。退職や異動のたびにゼロから引き継ぎ資料を作り直す負担は、中小企業にとって決して小さくありません。「オフボード」の [AI-BPO導入支援サービス](/hikitsugi-ai-bpo-roadmap) は、属人化した業務をAIと専任スタッフの体制で外部化・標準化し、人が抜けても業務が止まらない仕組みづくりを支援します。引き継ぎを「個人の頑張り」から「会社の仕組み」へ変えたい経営者の方は、ぜひご相談ください。

## よくある質問（FAQ）

### 業務引き継ぎチェックリストはいつ作り始めればよいですか？

退職・異動が決まった時点ですぐに着手してください。最終日が近づいてからでは、業務の棚卸しと引き継ぎ書作成が間に合わず、必ず漏れが生じます。逆算して期間を確保することが大切です。

### チェックリストはエクセルと紙のどちらがよいですか？

複数人で進捗を共有するならエクセルやスプレッドシートが便利です。完了欄を共有し、管理者がリアルタイムで確認できます。少人数で完結する場合は紙でも問題ありません。重要なのは形式より「完了の見える化」です。

### 引き継ぐ相手がまだ決まっていない場合はどうすればよいですか？

後任未定でも、まず引き継ぐ側が業務の棚卸しと引き継ぎ書の作成を進めてください。文書化しておけば、後任が決まり次第すぐに渡せます。後任不在のまま退職日を迎えると引き継ぎが完全に止まるため、文書化を優先しましょう。

### アカウントやパスワードの引き継ぎで注意することは？

平文での共有は避け、権限の付与・移管を正式な手順で行ってください。退職者のアカウントは放置せず、移管後に速やかに権限を削除します。アカウントの放置はセキュリティ事故の温床になります。

### チェックリストの項目は業種によって変えるべきですか？

はい。本記事の表は汎用的な項目です。自社特有の業務（許認可、専門システム、特定の取引先対応など）は項目を追加してください。一度自社版を作っておくと、次回以降の引き継ぎが格段に楽になります。

### 引き継ぎが完了したかどうかは誰が判断しますか？

管理者が判断します。引き継ぐ側と受け取る側だけに任せると認識のズレが残るため、第三者である管理者がチェックリストと引き継ぎ書を確認し、正式に完了を承認するプロセスを設けましょう。

## まとめ

- 業務引き継ぎチェックリストは、属人化を解消し抜け漏れを可視化する基本ツール
- 「引き継ぐ側」と「受け取る側・管理者」の双方でチェックし、漏れを二重に防ぐ
- 退職時はアカウント移管やセキュリティ、異動時は部署連携など、状況で重点が異なる
- チェックリストと引き継ぎ書はセットで使い、管理者が完了を正式に承認する

引き継ぎを個人任せにせず、会社の仕組みとして整えることが、人が抜けても止まらない組織づくりの第一歩です。
